2. MOCKITO::

👉 What is Mockito?

Mockito is a Java mocking framework used for testing.

It creates fake (mock) objects so that we can test a class without using real dependencies like:

Database

API

External service

👉 Mostly used with JUnit.


✅ Why Mockito is Needed?

Suppose:

Service Class → Repository → Database

While testing:

Database may not exist

Connection may fail

Testing becomes slow

✅ Mockito creates a fake Repository instead of real database.

Normal Situation
UserService → UserRepository → Database

During testing ❌ database not required.

Mockito does:

UserService → Mock Repository (Fake)

✅ Basic Mockito Annotations
| Annotation     | Purpose                 |
| -------------- | ----------------------- |
| `@Mock`        | Creates fake object     |
| `@InjectMocks` | Injects mock into class |
| `when()`       | Defines mock behavior   |
| `verify()`     | Checks method call      |


✅ Example (Very Important ⭐)
Service Class
class UserService {
    UserRepository repo;

    public String getUser() {
        return repo.findUser();
    }
}


Test Using Mockito + JUnit

import static org.mockito.Mockito.*;
import org.junit.jupiter.api.Test;
import org.mockito.InjectMocks;
import org.mockito.Mock;

class UserServiceTest {

    @Mock
    UserRepository repo;

    @InjectMocks
    UserService service;

    @Test
    void testUser() {

        when(repo.findUser())
            .thenReturn("Akhilesh");

        String result = service.getUser();

        assertEquals("Akhilesh", result);
    }
}

✅ No database used
✅ Fast testing
✅ Controlled output

✅ Important Mockito Methods
| Method         | Use                 |
| -------------- | ------------------- |
| `mock()`       | Create mock object  |
| `when()`       | Define behavior     |
| `thenReturn()` | Return fake value   |
| `verify()`     | Check method called |
| `times()`      | Check call count    |


Example:

verify(repo, times(1)).findUser();


✅ JUnit vs Mockito (Interview Question ⭐)
| JUnit             | Mockito               |
| ----------------- | --------------------- |
| Testing framework | Mocking framework     |
| Tests methods     | Creates fake objects  |
| Assertions        | Dependency simulation |

✅ JUnit 5 Library


✅ Step 5: Simple Mockito Test

import static org.mockito.Mockito.*;
import static org.junit.jupiter.api.Assertions.*;

import org.junit.jupiter.api.Test;

class DemoTest {

    @Test
    void testMock() {

        List list = mock(List.class);

        when(list.get(0)).thenReturn("Hello");

        assertEquals("Hello", list.get(0));
    }
}

Run → Run As → JUnit Test

✅ Method 2: Setup Mockito in Eclipse (With Maven)

Instead of manual jars → use Maven.

Just add dependency:

<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-core</artifactId>
    <version>5.11.0</version>
    <scope>test</scope>
</dependency>

✅ Automatically downloads everything.
